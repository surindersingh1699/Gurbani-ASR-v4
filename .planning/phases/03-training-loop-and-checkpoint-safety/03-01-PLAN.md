**3. SurtTrainer(Seq2SeqTrainer):**
Override `create_optimizer()` to build three AdamW parameter groups with discriminative learning rates.

CRITICAL implementation details:
- Use `id(param)` deduplication via a `seen_ids = set()` to handle the tied weight between `proj_out.weight` and `model.decoder.embed_tokens.weight`. Without this, the tied weight gets double gradient updates.
- Match parameter names using `"model.encoder" in name` for encoder, `"proj_out" in name` for proj_out (check BEFORE decoder since proj_out name doesn't contain "decoder"), and `"model.decoder" in name` for decoder.
- After building groups, verify total parameter count: `sum(len(g["params"]) for g in groups)` should be logged. Add a print statement: `[train] Parameter groups: encoder={N}, decoder={N}, proj_out={N}, total={N}`.
- Import LR constants from surt.config: ENCODER_LR, DECODER_LR, WEIGHT_DECAY.
- Return `self.optimizer` after creation.

Follow exactly the `build_param_groups` pattern from 03-RESEARCH.md (Pattern 3 / WhisperForConditionalGeneration Parameter Structure section).

**4. make_compute_metrics(processor):**
Factory function returning a `compute_metrics` callable:
- Replace -100 in `pred.label_ids` with `processor.tokenizer.pad_token_id`
- Decode both predictions and labels with `processor.tokenizer.batch_decode(ids, skip_special_tokens=True)`
- Compute WER as percentage: `100 * jiwer.wer(label_str, pred_str)`
- Return `{"wer": wer}`

**5. build_training_args():**
Function that creates and returns `Seq2SeqTrainingArguments` with all config constants:
- `output_dir=OUTPUT_DIR`
- `per_device_train_batch_size=BATCH_SIZE`
- `per_device_eval_batch_size=BATCH_SIZE`
- `gradient_accumulation_steps=GRAD_ACCUM`
- `max_steps=MAX_STEPS`
- `warmup_steps=WARMUP_STEPS`
- `learning_rate=LEARNING_RATE` (base LR, overridden by create_optimizer groups)
- `lr_scheduler_type="cosine"`
- `weight_decay=WEIGHT_DECAY`
- `bf16=True`
- `gradient_checkpointing=True`
- `gradient_checkpointing_kwargs={"use_reentrant": False}`
- `eval_strategy="steps"`
- `eval_steps=EVAL_STEPS`
- `save_strategy="steps"`
- `save_steps=SAVE_STEPS`
- `save_total_limit=SAVE_TOTAL_LIMIT`
- `predict_with_generate=True`
- `generation_max_length=GENERATION_MAX_LENGTH`
- `logging_steps=25`
- `dataloader_num_workers=0` (required for streaming IterableDataset)
- `load_best_model_at_end=False` (we handle best model via custom callback)
- `push_to_hub=False` (we handle Hub push via custom callback)
- `report_to="none"` (no TensorBoard/W&B for v1)
- `remove_unused_columns=False` (streaming datasets may not have column_names)

Do NOT use `evaluation_strategy` (renamed to `eval_strategy` in transformers v5).
Do NOT use `fp16=True` (use `bf16=True` for Ampere GPUs).
Do NOT set `num_train_epochs` (streaming datasets have no length -- use max_steps).

**Pattern notes:**

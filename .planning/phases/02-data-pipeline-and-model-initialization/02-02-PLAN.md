<objective>
Create `surt/data.py` with the streaming data pipeline that loads Gurbani audio from HuggingFace Hub, applies waveform augmentation to training data only, materializes a fixed validation subset, and provides a data collator that correctly masks pad tokens to -100 with double-BOS protection.

Purpose: The data pipeline is the second highest-risk component after model init. Every piece must produce outputs that exactly match what Whisper expects -- 80-bin log-Mel spectrograms, Gurmukhi BPE tokens with language prefix, -100 masked padding. Getting any of these wrong produces silently corrupt training data.

Output: `surt/data.py` with `get_train_dataset()`, `get_val_dataset()`, and `DataCollatorSpeechSeq2SeqWithPadding`.

  <action>
Create `surt/data.py` with the following components. The module accepts a processor (WhisperProcessor) as a parameter rather than importing it directly -- this keeps the dependency direction clean and testable.

**Module constants:**
- `VAL_SIZE = 300` -- number of validation examples to materialize
- `AUDIO_COLUMN = "audio"` -- configurable column name (default for most ASR datasets)
- `TEXT_COLUMN = "sentence"` -- configurable column name (Common Voice convention; can be overridden)

**1. Augmentation pipeline (module-level):**
```python
from audiomentations import Compose, AddGaussianNoise, RoomSimulator, TimeStretch, PitchShift

augment = Compose([
    AddGaussianNoise(min_amplitude=0.001, max_amplitude=0.015, p=0.4),
    RoomSimulator(p=0.3),
    TimeStretch(min_rate=0.8, max_rate=1.25, p=0.2),
    PitchShift(min_semitones=-4, max_semitones=4, p=0.2),
])
```

**2. `get_train_dataset(dataset_name: str, processor: WhisperProcessor, split: str = "train", text_column: str = "sentence") -> IterableDataset`:**

- Load dataset with `load_dataset(dataset_name, split=split, streaming=True)`
- Cast audio column to 16kHz: `.cast_column("audio", Audio(sampling_rate=16000))`
- Shuffle with `seed=42, buffer_size=500`
- Apply `.map()` with an inner `prepare_train` function that:
  - Gets raw audio array as float32
  - Applies `augment(samples=samples, sample_rate=16000)` to waveform BEFORE feature extraction
  - Extracts log-Mel features via `processor.feature_extractor(samples, sampling_rate=16000).input_features[0]`
  - Tokenizes text via `processor.tokenizer(example[text_column]).input_ids`
  - Returns dict with `input_features` and `labels` keys only
- Use `remove_columns` to strip all original columns (streaming IterableDataset may not have `.column_names` -- handle gracefully by passing column list or omitting if unavailable)
- Return the mapped IterableDataset
- Print: `[data] Training dataset streaming from {dataset_name} (split={split})`

**CRITICAL:** Augmentation MUST be applied to raw waveform BEFORE feature extraction. The order is: audio["array"] -> augment -> feature_extractor. Never apply augmentation to log-Mel features.

**3. `get_val_dataset(dataset_name: str, processor: WhisperProcessor, split: str = "validation", text_column: str = "sentence", val_size: int = 300) -> Dataset`:**

- Load dataset with `load_dataset(dataset_name, split=split, streaming=True)`
- Cast audio column to 16kHz: `.cast_column("audio", Audio(sampling_rate=16000))`
- Take first `val_size` examples: `list(stream.take(val_size))`
- Materialize into regular Dataset: `Dataset.from_list(val_examples)`
  - Fallback if `from_list` unavailable: `Dataset.from_dict({k: [ex[k] for ex in val_examples] for k in val_examples[0]})`
- Apply `.map()` with an inner `prepare_val` function that:
  - Gets raw audio array (NO augmentation)
  - Extracts log-Mel features via `processor.feature_extractor(audio["array"], sampling_rate=audio["sampling_rate"]).input_features[0]`
  - Tokenizes text via `processor.tokenizer(example[text_column]).input_ids`
  - Returns dict with `input_features` and `labels` keys only
- Use `remove_columns=val_dataset.column_names`
- Print: `[data] Validation dataset: {len(val_dataset)} examples materialized in memory`
- Return the mapped Dataset

**CRITICAL:** NO augmentation on validation data. Use a separate prepare function.

**4. `DataCollatorSpeechSeq2SeqWithPadding` dataclass:**

- Fields: `processor: Any`, `decoder_start_token_id: int`
- `__call__(self, features: list[dict]) -> dict`:
  - Separate input_features and labels from features
  - Pad input_features using `self.processor.feature_extractor.pad(input_features, return_tensors="pt")`
  - Pad labels using `self.processor.tokenizer.pad(label_features, return_tensors="pt")`
  - Mask padding tokens to -100: `labels.masked_fill(attention_mask.ne(1), -100)`
  - **Double-BOS guard:** If ALL labels in batch start with `decoder_start_token_id`, strip the first column: `labels = labels[:, 1:]`
  - Set `batch["labels"] = labels`
  - Return batch

**CRITICAL anti-patterns to avoid (from research):**
- Do NOT use `num_proc > 0` with streaming IterableDataset -- not supported
- Do NOT apply augmentation inside the data collator -- must happen on raw waveform
- Do NOT use a single prepare_dataset for both train and val -- augmentation would leak
- Do NOT forget the double-BOS strip -- causes garbled output
  </action>

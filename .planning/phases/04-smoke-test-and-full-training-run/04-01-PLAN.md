# Phase 4 Plan 01

## Goal

Implement all missing Phase 4 code paths so the pipeline can run pre-flight checks, execute a 10-step smoke run, and then complete a full run with final push to the production Hub repo.

## Requirements

- TEST-01
- TEST-02
- TEST-03
- CKPT-04

## Tasks

1. Add a new `surt/smoke_test.py` module.
2. Implement `run_generation_preflight()` for Gurmukhi output sanity.
3. Implement `run_batch_preflight()` for 16kHz, `-100` masking, BOS validation.
4. Refactor `surt/train.py` to support multiple execution modes (`full`, `smoke`, `phase4`).
5. Add `run_training_job()` helper to avoid duplicated trainer wiring.
6. Add smoke validation check (`validate_smoke_training`) for loss and LR behavior.
7. Add final push function (`push_final_model_to_hub`) that targets `HF_MODEL_REPO`.
8. Update roadmap and state docs so decisions are reflected across phases.
9. Run local static validation (`python3 -m py_compile surt/train.py surt/smoke_test.py`).

## Notes

- Runtime completion of the full run is intentionally deferred to RunPod.
- This plan completes implementation and orchestration; it does not claim remote training has already finished.
